# deployment.md

## 1. 문서 목적

본 문서는 Java daemon의 패키징, 배포, 실행, 운영에 대한 기준을 정의한다.

본 문서의 목적은 다음과 같다.

- 개발 결과물을 어떤 형태로 배포할지 명확히 한다.
- Linux 서버에서 daemon을 실행하는 기본 절차를 정리한다.
- 배포 시 필요한 파일과 구성요소를 정의한다.
- 로컬 실행과 운영 환경 실행을 구분한다.
- 운영 과정에서 확인해야 할 기본 사항을 정리한다.

본 문서는 배포 및 실행 기준 문서이며, 기능 요구사항이나 Agent 역할을 정의하지 않는다.

---

## 2. 적용 범위

본 문서는 본 repository에서 개발되는 Java daemon의 배포 및 실행에 적용된다.

적용 대상은 다음과 같다.

- Java source code의 빌드 결과물
- 실행 스크립트
- 설정 파일
- Linux 서버 배치 구조
- 로컬 검증 실행
- 운영 환경 배포 및 실행 방식

본 문서는 Agent 실행 계층 자체가 아니라, Agent가 개발하는 **제품 계층의 daemon**을 대상으로 한다.

---

## 3. 배포 대상

현재 배포 대상은 Linux 서버에서 동작하는 Java daemon이다.

배포 대상의 기본 특성은 다음과 같다.

- Java 기반 실행 파일(jar) 형태로 패키징한다.
- 장기 실행을 전제로 한다.
- 운영형 소프트웨어로 취급한다.
- 시작, 종료, 로그, 재실행 가능성을 고려한다.

---

## 4. 배포 산출물

배포 시 기본 산출물은 다음과 같이 구성한다.

### 4.1 필수 산출물

- 실행 가능한 jar 파일
- 실행 스크립트
- 필요한 설정 파일
- 배포 버전 식별 정보

### 4.2 권장 산출물

- 배포 요약 문서 또는 릴리스 노트
- 기본 로그 경로 안내
- 실행 예시 명령
- 검증 절차 요약

### 4.3 현재 기준 예시

현재 기준 배포 산출물 예시는 다음과 같다.

```text
dist/
├─ daemon-app.jar
├─ run-daemon.sh
└─ config/

## 5. 빌드 기준
Java daemon은 빌드 도구를 통해 패키징한다.

현재 기준 선택지는 다음과 같다.

- Maven (pom.xml)
- Gradle (build.gradle)

현재 저장소 예시는 Maven 기준을 우선 사용한다.

빌드 결과는 다음 조건을 만족해야 한다.

- Java 컴파일 가능
- 테스트 실행 가능
- jar 패키징 가능
- 실행 시작 클래스 지정 가능

## 6. 로컬 실행 기준
로컬 실행은 개발 및 기본 검증을 위한 실행 방식이다.

### 6.1 목적
- 코드가 기본적으로 실행 가능한지 확인한다.
- 빠른 기능 검증을 수행한다.
- 패키징 전 기본 동작을 점검한다.

### 6.2 현재 기준 방식
로컬 실행은 scripts/run-local.sh를 통해 수행한다.

이 스크립트는 다음 흐름을 따른다.

- 테스트 수행
- 패키징 수행
- 생성된 jar 실행

### 6.3 전제 조건
로컬 실행 환경에는 다음이 준비되어 있어야 한다.

- Java 런타임
- 빌드 도구(Maven 또는 Gradle)
- repository 내 pom.xml 또는 build.gradle
- 실행 가능한 main class

## 7. 패키징 기준
패키징은 배포 가능한 산출물을 만드는 절차이다.

### 7.1 목적
- 실행 파일을 일관된 형식으로 생성한다.
- 운영 환경에 전달 가능한 구조를 만든다.
- 실행 스크립트와 함께 배포 단위를 구성한다.

### 7.2 현재 기준 방식
- 패키징은 scripts/package-daemon.sh 를 통해 수행한다.

현재 기준 패키징 흐름은 다음과 같다.

- 테스트 수행
- 빌드 수행
- jar 파일 생성
- dist/ 디렉터리 구성
- 실행 스크립트 생성 또는 복사

### 7.3 패키징 결과 예시

dist/
├─ daemon-app.jar
├─ run-daemon.sh
└─ config/

## 8. Linux 서버 배포 기준
배포는 생성된 산출물을 Linux 서버에 배치하는 절차를 의미한다.

### 8.1 기본 원칙

- 배포 산출물은 버전 식별이 가능해야 한다.
- 실행 파일과 설정 파일은 분리 가능해야 한다.
- 운영 서버에 불필요한 개발 파일은 포함하지 않는다.
- 실행 경로와 로그 경로는 명확해야 한다.

### 8.2 배치 구성 예시

/opt/my-daemon/
├─ current/
│      ├─ daemon-app.jar
│      ├─ run-daemon.sh
│      └─ config/
├─ releases/
└─ logs/

위 예시는 기준 예시이며, 실제 서버 구조는 환경에 따라 조정할 수 있다.

### 8.3 배포 절차 개요

기본 배포 절차는 다음과 같다.

1. 패키징 결과 확인
2. 서버 대상 경로 확인
3. 산출물 업로드 또는 복사
4. 실행 스크립트 권한 확인
5.설정 파일 확인
6. daemon 실행 또는 재시작
7. 로그 및 상태 확인


## 9. 실행 기준
### 9.1 실행 방식

Java daemon은 기본적으로 jar 실행 방식으로 동작한다.

예시:

     </> Bash
     java -jar daemon-app.jar

실제 실행은 편의와 일관성을 위해 스크립트를 통해 수행한다.

예시:
     </> Bash
     ./run-daemon.sh

### 9.2 실행 시 확인 항목

실행 전 또는 실행 직후 아래 항목을 확인한다.

- Java 버전
- 설정 파일 존재 여부
- 로그 경로 존재 여부
- 포트 또는 파일 경로 충돌 여부
- 필수 디렉터리 존재 여부
- 예외 발생 여부

## 10. 운영 기준

Java daemon은 운영형 소프트웨어로 취급하므로 다음 기준을 고려한다.

### 10.1 장기 실행 안정성
- 일시적 성공이 아니라 지속 실행 가능성을 확인한다.
- 반복 실행 또는 재시작 시 동작 일관성을 유지해야 한다.

### 10.2 로그 및 추적성
- 시작 로그, 종료 로그, 예외 로그가 확인 가능해야 한다.
- 문제 발생 시 최소한의 원인 추적이 가능해야 한다.

### 10.3 예외 처리
- 예외 발생 시 비정상 종료 여부를 확인할 수 있어야 한다.
- 치명적 오류와 일반 오류를 가능한 한 구분한다.

### 10.4 설정 관리
- 설정값은 코드와 분리 가능한 구조를 지향한다.
- 환경별 차이가 발생할 수 있는 값은 외부화 가능성을 고려한다.

### 10.5 재시작 가능성
- 프로세스 중단 후 재실행 가능해야 한다.
- 재실행 시 기본 초기화 흐름이 명확해야 한다.

## 11. 운영 환경 실행 방식

운영 환경에서는 필요 시 다음 방식 중 하나를 사용할 수 있다.

- 직접 스크립트 실행
- nohup 기반 백그라운드 실행
- systemd 서비스 등록

현재 단계에서는 직접 실행 또는 스크립트 기반 실행을 기본으로 하며, 실제 운영 단계에서 systemd 적용 여부를 결정한다.

## 12. 검증 기준

배포 또는 실행 후 최소한 다음 항목을 검증한다.

- jar 파일이 정상 생성되었는가
- 실행 스크립트가 정상 동작하는가
- 프로세스가 정상 시작되는가
- 초기 예외가 없는가
- 로그가 생성되는가
- 기본 기능이 최소 수준으로 응답하는가

테스트 및 검증 결과는 필요 시 eval/ 또는 작업 결과 기록에 반영한다.

13. 현재 미정 사항

다음 항목은 현재 미정 상태이다.

- 운영 서버의 최종 디렉터리 정책
- systemd 적용 여부
- 실제 로그 파일 경로 정책
- 설정 파일 형식
- 운영 배포 자동화 적용 여부
- 릴리스 버전 관리 방식
- 백업 및 롤백 방식

위 항목은 실제 운영 환경과 기능 요구사항이 정리되는 단계에서 확정한다.

## 14. 배포 관련 리스크 및 주의 사항

### 14.1 개발 환경과 운영 환경 차이

로컬에서는 정상 실행되더라도 운영 서버 환경 차이로 문제가 발생할 수 있다.

### 14.2 설정 누락

설정 파일, 경로, 권한 등이 누락되면 실행 실패가 발생할 수 있다.

### 14.3 로그 미비

로그가 부족하면 운영 중 문제 추적이 어려워질 수 있다.

### 14.4 실행 스크립트 의존성

스크립트가 특정 경로 또는 환경 변수에 과도하게 의존하면 재사용성이 낮아질 수 있다.

### 14.5 운영 방식 미확정

현재는 기본 실행 기준만 정의되어 있으며, 실제 서비스 운영 방식은 후속 확정이 필요하다.

## 15. 문서 유지 원칙

다음 경우 본 문서를 갱신한다.

- 빌드 도구가 변경된 경우
- 패키징 구조가 변경된 경우
- 실행 스크립트 방식이 변경된 경우
- 운영 서버 배치 구조가 변경된 경우
- systemd 등 운영 방식이 확정된 경우
- 로그 및 설정 정책이 변경된 경우

본 문서는 배포 및 실행 기준 문서이므로, 실제 배포 방식이 바뀌면 즉시 함께 수정한다.
